The architecture avoids the heavy RTT penalty of ARQ by relying entirely on Forward Error Correction (FEC). To remain under the strict 2.0x bandwidth limit, the sender computes an XOR parity block between payloads `seq - 1` and `seq - 3` and appends it to 95% of packets. The receiver iteratively chains these parity blocks to instantly recover from random drops and burst losses up to length 2 without requiring explicit playout scheduling. We grade at `delay_ms = 140`, which allows the 60ms maximum FEC recovery window to safely span Profile B's 80ms network latency. The system will break if consecutive burst losses exceed 3 packets or if network jitter sustains above 140ms.
